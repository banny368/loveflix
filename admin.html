🔥 Firebase Configuration
📖 How do I get my Firebase config? (step-by-step)
  1. Go to console.firebase.google.com and create a project (e.g. "loveflix").
  2. Authentication → Sign-in method: enable Email/Password, then Users → Add user (your admin email + password).
  3. Firestore Database → Create database (start in test mode).
  4. In Firestore → Rules, paste: 
    rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{collection}/{document} {
      allow read: if true;
      allow write: if request.auth != null;
    }
  }
}
  5. Project Settings → General → Your apps → Web app → register an app and copy the firebaseConfig block.
  6. Paste that whole block in the box below and click Parse, then Save.

⚠️ Saved here = this browser only. To make it permanent for all devices and visitors, put the same values in js/config.js and re-upload the site.

☁️ Cloudinary Configuration
📖 How do I set up Cloudinary? (step-by-step)
  1. Create a free account at cloudinary.com.
  2. On the Dashboard, copy your Cloud Name.
  3. Go to Settings → Upload → Upload presets → Add upload preset.
  4. Set Signing Mode to Unsigned, optionally set the folder to loveflix, and save.
  5. Enter the Cloud Name and preset name below, click Test, then Save.

💡 Cloudinary settings also sync to your database, so they work on every device you sign in from. For visitors nothing changes — they only view the already-uploaded media.

🛡️ Database Rules

If saving anything shows "Missing or insufficient permissions", your Firestore security rules are blocking writes (Firebase test mode expires after 30 days). Fix it in one minute below.

📖 Fix in 4 steps
  1. Click 📋 Copy Rules below.
  2. Click 🚀 Open Rules Editor — it opens your exact Firebase project's rules page.
  3. Select everything in the editor, paste, and press Publish.
  4. Come back here and click 🧪 Test Permissions — both pills should turn green.
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{collection}/{document} {
      allow read: if true;
      allow write: if request.auth != null;
    }
  }
}

These rules let anyone view your LoveFlix (needed for your partner to see it) but only you (signed-in admin) can change anything.
